# シュナウザー・アジリティ・フラップ — 設計書 (v2)

## 1. コンセプト転換

v1（タイミングよくタップして障害物を通過）から、**Flappy Bird 型の高さ管理ゲーム**へ全面変更する。

- 操作は **タップ（クリック / スペース）のみ**。
- **タップするたびに上昇（フラップ）、放置すると重力で落下**。
- 障害物は「タイミングで避ける」のではなく、**今この瞬間に正しい高さにいるか**で当たり判定する。
- コースは **横からの視点（サイドビュー）の標高プロファイル**。緑＝触れてOK、赤＝ミス。

## 2. 画面構成（スマホ縦持ち）

```
┌─────────────────────────┐
│ 上：コース先読み（サイドビュー・引き）  │  画面の ~42%
│   緑/赤の地形を遠くまで表示。          │  「上を見て予測」
│   ● 犬の現在地マーカー / NOW 縦線。     │
│   - - 安全ルートのガイド線（緑点線）。   │
├─────────────────────────┤
│ 下：プレイ（サイドビュー・寄り）        │  画面の ~58%
│   犬が大きい(約2倍)・縦に窮屈・横の視界が狭い │  「窮屈・高速・スリル」
│   スクロール3倍速。フラップで高さ調整。    │
└─────────────────────────┘
```

- **上下とも同じサイドビュー**。上は引き（遠くまで・小さく）、下は寄り（大きく・狭く）。
- 下画面は**視認が悪い**＝1障害物ぶんくらいしか先が見えない。だから上で予測しておく必要がある。

## 3. 当たり判定モデル（緑＝安全 / 赤＝ミス）

フィールドを縦 `FH`（フィールド単位）とし、地面 `floorY = FH`。犬は半径 `R` の円。
各障害物は「下からの赤」「上からの赤」を持ち、その**隙間（緑）＝通れる安全帯**になる。

| 区分 | 説明 | 当たり |
|------|------|--------|
| **歩いてOK（緑の地面）** | フラップしなければ着地して休める。落ちても問題ない区間。 | 地面=緑・安全 |
| **ジャンプで避ける（赤）** | 地面から赤いバーがせり上がる。**上を越える**。 | 赤に触れる=ミス |
| **ちょうどの高さを保つ（赤上下＋緑中央）** | 上下が赤、中央だけ緑の通り穴。**高すぎても低すぎてもミス**。 | 赤に触れる=ミス |

統一表現：各障害物は
- `redLow`：地面から上方向 `yLowTop` までの赤（＝それより上にいないとミス）
- `redHigh`：天井から下方向 `yHighBottom` までの赤（＝それより下にいないとミス）
- 緑の安全帯 `gap = [yHighBottom, yLowTop]`

をオプションで持つ。3種目はこの組み合わせで表現できる。

## 4. アジリティ種目への割り当てと内容

| 種目 | モチーフ | redLow | redHigh | 操作 |
|------|----------|--------|---------|------|
| **直線走 / 休憩 (Run)** | スタート〜直線 | なし | なし | 放置で着地・休める安全地帯 |
| **ハードル (Hurdle)** | ジャンプ障害 | あり（バー高 `barH`） | なし | フラップで飛び越える |
| **タイヤ (Tire)** | 輪くぐり | あり | あり | 中央の緑をちょうどの高さで通す |
| **トンネル (Tunnel)** | 筒くぐり | なし | あり（天井） | 低空を維持して潜る |
| （発展）**Aフレーム** | 登り障害 | — | — | 緑の高台に飛び乗って渡る（v2.1候補） |

色：赤い部分（`redLow`/`redHigh`）＝赤。地面・安全帯＝緑。

## 5. 難易度の配分（距離で段階的に難化）

| 距離 | 出現比率（重み） | ギャップ/間隔 |
|------|------------------|---------------|
| 0–800m | Run 3 / Hurdle 3 / Tire 1 / Tunnel 1。バー低め。 | 広い・ゆったり |
| 800–2000m | Run 1 / Hurdle 3 / Tire 2 / Tunnel 2。 | やや詰まる・緑帯やや狭く |
| 2000m+ | Hurdle 3 / Tire 3 / Tunnel 3（Run少）。連続。 | 狭い・高速で連発 |

- 進むほど：スクロール速度＋、障害物間隔−、緑帯（gap）−、バー高＋。
- 序盤は「休憩(緑)」を多めにして操作に慣れさせ、終盤は赤を増やして窮屈に。

## 6. ライフ / スコア

- **3ミスでゲームオーバー**（ライフ3）。
- **スコア＝距離（m）**。タイマー無しのエンドレス。
- ミス時：ライフ−1、約1.2秒の無敵点滅、安全帯へスナップ復帰、画面シェイク＋効果音。スクロールは止めない（テンポ維持）。
- ベスト距離は `localStorage` 保存。

## 7. 物理パラメータ（初期値・調整可能）

| 名前 | 値 | 意味 |
|------|----|------|
| `FH` | 200 | フィールド高（縦・フィールド単位） |
| `R` | 26 | 犬の当たり半径（≒画面で約2倍の大きさ感） |
| `GRAVITY` | 760 | 重力加速度 (units/s²) |
| `FLAP_V` | -255 | フラップ時の上昇初速 |
| `MAX_FALL` | 430 | 落下速度の上限 |
| `SCROLL` | 720→900 | スクロール速度 px/s（v1の約3倍、距離で微増） |
| `PX_PER_METER` | 30 | 距離換算 |

ギャップ：Tire 96→78、Tunnel 天井クリアランス 95→80、Hurdle バー高 50→110。

## 8. 実装方針

- 単一 `dog/index.html`（依存ゼロ）。Canvas + Vanilla JS。
- ワールドは横=worldX(px)、縦=field単位。障害物は等間隔生成（先読み分を先に生成）。
- 描画は上(引き)・下(寄り)で同じワールドを別スケールで2回描く共通関数。
- 当たり判定：犬の円 vs 各障害物の赤矩形（x重なり範囲のみ）。無敵中は無視。
- 入力：`pointerdown` / Space でフラップ。スタート/ゲームオーバーは DOM オーバーレイ。
- 音：Web Audio 合成（ファイル不要）。フラップ・ミス・ゲームオーバー音。
