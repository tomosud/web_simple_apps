# 16茶ゲーム リニューアル計画・経緯

## 初期分析（試作版の問題点）

試作版 `tea_game.html` を分析した結果、以下の課題を特定:

### ゲーム性の問題
1. **推理が薄い**: 各色の値が隠されているが、高さの差分を暗算する必要がある。面倒なだけで面白くない
2. **「ちょっと飲む」がランダム(1-3)**: 戦略ゲームなのに核心の調整手段が運任せ
3. **難易度が一定**: 面が進んでも開始高さ1-15のランダム、手数は常に4。単調になる
4. **フィードバック不足**: 値を推理した達成感がない。失敗時に何が正解だったか不明

### UI/UXの問題
5. **PCレイアウト**: ボトルと情報パネルが横並び。スマホで見ると崩れる
6. **ボタンが小さい**: モバイルでのタップがしづらい
7. **テンポが遅い**: クリア後3秒待ち、失敗後3秒待ち

## 設計方針

### 核心コンセプト
> **「探索(情報を得る)」と「勝負(情報を使う)」のジレンマ**

序盤はリスク低く色を試して値を学び、終盤は得た知識で16ぴったりを狙う。
面が進むほど探索の余裕がなくなり、不完全な情報で勝負する緊張感が増す。

### 操作の制約
ユーザー要望: **ボタンを押す以外の操作はしたくない**
→ メモ機能等は不要。値の自動表示で推理の可視化を実現

## 改善内容の詳細

### 1. 値の自動表示（最重要改善）
- ボタンに初期状態「?」を表示
- 押した瞬間、差分から値を計算して「+3」等を自動表示
- 推理の面倒な部分を自動化し、**判断に集中させる**
- レベル終了時に全色の値を答え合わせ表示 → 「あーそうだったのか！」の悔しさ・納得感

### 2. 難易度カーブ
| レベル | 開始高さ | 手数 | 体験 |
|--------|----------|------|------|
| 1-3 | 2~6 | 6 | のんびり探索 |
| 4-6 | 6~10 | 5 | 無駄打ち禁止 |
| 7-9 | 10~13 | 4 | 1手目から悩む |
| 10+ | 13~15 | 3 | ほぼ一発勝負 |

### 3. 「ひとくち」固定値化
ランダム(1-3) → **常に1だけ減少**
- 「4を足して1飲む = 実質+3」という技が使える
- 戦略に組み込める確定アクション

### 4. 演出強化
- 高さ12超: ボトル背景が暖色に
- 高さ14超: 脈動アニメーション
- 高さ15: 強い脈動
- テンポ高速化: 演出1.5秒→即リトライ

### 5. モチベーション維持
- 連勝ストリーク表示
- 称号システム（面数に応じて自動付与）
- ハイスコア（最高面数）のlocalStorage保存

### 6. モバイルファーストUI
- 縦1カラム、スクロール不要
- ボトル中央に大きく
- ボタンは画面下部に大きく配置

## 技術方針
- 単一HTML（CSS + JS埋め込み）を維持
- 外部依存なし → GitHub Pages対応
- CSS変数で演出制御
- localStorage でハイスコア・称号保存

## 実装・テスト結果

### 実装完了項目
全項目を `tea_game.html` に実装完了。

### テスト結果（Puppeteer + モバイルビューポート 375x812）
1. **レイヤー表示**: 初回実装で `.tea-layers` に `top:0` がなく、子要素の%指定が効かなかった → absolute positioning方式に修正して解決
2. **値の自動表示**: ボタン押下で「?」→「+2」等に変化。revealアニメーションも動作
3. **ひとくち飲む**: 正確に-1。上から順(黄→青→緑→赤)に減少
4. **クリアフロー**: 16到達 → オーバーレイ表示(答え合わせ) → 1.5秒後に次の面へ自動遷移
5. **バーストフロー**: 16超過 → オーバーレイ表示 → 1.8秒後に面1からリスタート
6. **テンション演出**: 高さ12超でボトル枠がオレンジに光る。動作確認済み
7. **ストリーク**: 連勝で「🔥2連勝」表示、失敗でリセット
8. **称号/ハイスコア**: localStorage保存・表示とも正常
9. **モバイルUI**: 1カラム縦レイアウト、大きなボタン、スクロール不要

### 修正したバグ
1. **レイヤー描画**: `.tea-layers` を flex → absolute positioning 方式に変更。各レイヤーを `bottom` と `height` の%で配置
2. **モバイルレイアウト**: ボトル `max-height` を 340px→200px に縮小、各要素のpadding/marginを圧縮し、`.main` を `justify-content: flex-end` にしてボタンとの隙間を解消。375x812 viewport で全要素が1画面に収まることを確認
